# 项目状态

## Current Milestone

当前项目是一个轻量化专项维修资金审计辅助 Demo。当前里程碑聚焦单项目材料上传、规则审计输出、证据化前端展示，以及本地 LLM 辅助字段归类。

## System Capabilities

- 前端基于 React + Vite，提供实时审计、历史报告和演示版政策库页面。
- 后端基于 FastAPI，提供 `/api/audit-engine/files/analyze-single` 多文件上传审计接口。
- Excel 文件参与审计解析；PDF 和其他非审计输入仅作为附件保留和展示。
- 审计流程会将来源数据映射为标准字段，执行分层规则校验，并生成 `audit_view` 供前端展示。
- 本地 LLM 可在配置可用时辅助语义字段归类，但最终审计结论仍由规则引擎输出。
- 前端可展示合规维度摘要、AI 审计综合意见、问题卡片、项目概览、证据区、法规匹配、时间线和审计人员视图。

## Recent Changes

- 合规维度摘要已调整为旧版卡片结构和风险色风格，同时保留当前维度文本。
- AI 审计综合意见下的 `next_actions` 已改为独立分区，并按一行一条建议展示。
- 审计展示拆分为可复用的 audit-view panels 组件。
- 审计引擎后端已迁移到当前轻量化 Demo 栈。
- 已接入本地 LLM 模型检查、失败回退和超时配置。
- 已优化审计矩阵、证据展示和审计人员视图。

## Decisions

- 本项目定位为审计辅助工具，不替代人工审计结论。
- 当前 Demo 不包含数据库、权限系统或 RAG 能力。
- 当前阶段只有 Excel 作为有效审计输入；PDF 仅作为附件展示。
- 缺失材料必须明确标注，不伪造、不补写不存在的证据。
- parser / 规则字段与 LLM 结果冲突时，保守采用 parser / 规则字段作为最终值。
- `PROJECT_STATE.md` 是状态快照，不作为变更日志使用。

## Known Limitations

- PDF/OCR 解析尚未实现。
- 如果解析到多个候选项目，当前只审计第一个项目，并提示其余项目需要人工复核。
- 政策库上传目前只是 localStorage 演示，不参与审计引擎。
- 本地 LLM 依赖外部 LM Studio 或 OpenAI-compatible 服务配置。
- 当前 CORS 配置只显式允许 `localhost:3000` 和 `127.0.0.1:3000`。

## Next Steps

- 在 Excel 审计流程稳定后，再扩展 PDF/OCR 解析。
- 决定政策库上传是否需要进入可检索审计依据链路。
- 为上传、审计视图映射和关键展示状态补充前后端回归测试。
